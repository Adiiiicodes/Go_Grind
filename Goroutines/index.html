🎯 How Goroutines Work

The Loop:

for i := 0; i < 1000000; i++ {
    go func(n int) {
        fmt.Println("i: ", n)
    }(i)
}

Step-by-step Execution:

Iteration 1 (i=0):
├── Start goroutine G0 → [G0] runs separately
└── Continue loop immediately (DON'T wait for G0)

Iteration 2 (i=1):
├── Start goroutine G1 → [G1] runs separately
└── Continue loop immediately (DON'T wait for G1)

Iteration 3 (i=2):
├── Start goroutine G2 → [G2] runs separately
└── Continue loop immediately (DON'T wait for G2)

The Key Concept:

⚡ THE LOOP FINISHES IN MILLISECONDS! ⚡
All 1,000,000 goroutines are now running CONCURRENTLY

[G0] [G1] [G2] [G3] [G4] [G5] ... [G999999]
ALL FIGHTING TO PRINT TO TERMINAL

The "Detach" Concept:

go func(n int) { fmt.Println("i: ", n) }(i)

THE LOOP DOESN'T WAIT HERE!
It immediately continues to the next iteration.

Main: "Hey G0, go print '0'"
Main: "Hey G1, go print '1'" ← Doesn't wait for G0
Main: "Hey G2, go print '2'" ← Doesn't wait for G1

Why The Order Is Random:

GO SCHEDULER (The Traffic Cop)

• Decides which goroutine runs on which thread
• Decides when each goroutine gets CPU time
• Decides the execution order

G5 → G0 → G3 → G7 → G1 → G8 → G2 → G4 → G6 → G9
↑ RANDOM ORDER EVERY TIME! ↑
